# 报工周报表单字段事实源（逆向自 2026-07-21 导出文件）

来源：宜搭数据管理页「导出」产物 `报工周报_YYYY-MM-DD-*.xlsx`，第 1 行是**宜搭组件 ID**、
第 2 行是字段名——这直接解决了 P3（OpenAPI 直提）的 `formDataJson` 组件 ID 问题，
也是 P2（Playwright）字段定位与合法取值的事实源。

## 主表字段 → 组件 ID

| 组件 ID | 字段 |
|---|---|
| `F0000001` | 报工开始日期 |
| `F0000002` | 报工结束日期 |
| `F0000013` | 周总工时长统计（小时）（公式，勿写） |
| `F0000043` | 周有效工作时长统计（小时）（公式，勿写） |
| `F0000062` | 工作计划及完成情况（附件） |
| `F0000068` | 本周特殊情况说明 |

## 实际取值（来自历史已提交周报，非猜测）

- **项目/产品名称 与 项目/产品**（两个独立字段，值相同）：
  `D-PD-26002    工业数据智能标注平台 v3.0    王晓明`
  ——注意：**不是**附件 xlsx 里的「D-DP-25002 工智酷博」；表单选的是标注平台产品线，
  附件「所关联的项目/活动」写工智酷博，两处编号不同是常态。
- **项目类型**：`产品研发`（开发/项目会议行）、`公司和部门运营活动`（周例会/休假行，此时项目名可空）
- **工作状态**：`算法开发` / `内部会议` / `内外部会议` / `休假` / `其它`（差旅途中等）
- **星期**：`星期一` … `星期日`
- **填报风格**：一天**多行**（站会 0.5h + 开发主行 + 临时会议行），工时可带小数
  （0.5/1.5/7.5/11/12），周总工时 40-55h 常见；休假行=状态「休假」8h、
  项目类型「公司和部门运营活动」、无项目无内容。

## P2 真机联调发现（2026-07-21，全链路已通）

- 表单在 `iframe`（URL 含 `FormAdapter`）内渲染；列表页点「新增」后等 frame 内出现「报工开始日期」再操作。
- **字段编码直接是控件 DOM id**（`#F0000001`…）；子表容器 id=`2ee34a58…`（以数字开头，选择器必须用 `[id="…"]` 不能用 `#`）。
- 子表行 = `[id=子表] .ant-spin-container > .subgrid-sheet__row`。**坑：行内滚动容器与行同名 class**，直接匹配 `.subgrid-sheet__row` 数量翻倍、偶数行会覆盖上一行。
- 日期 = ant-calendar readonly input：点开 → `.ant-calendar-input` 输日期 + Enter；报工结束日期/星期/负责人/周总工时均服务端联动自动带出。
- 下拉 = h3-dropdown(ant-select)：点开后**按文本从后往前找可见项**点击——同名选项会以隐藏 li 残留在先前行的菜单里。
- 「项目/产品名称」关联下拉在孤立探测时曾"无数据"（前端只拉查询配置不拉行数据），但在完整填表流程中正常出数据并可选中——按正常流程填即可。
- 附件 `input[type=file]` set_input_files 直接可用（上传即到服务端临时区）。
- 底部按钮文本带空格：「暂 存」/「提 交」。
- 「主要工作内容」单元格上限 **200 字**（控件带 n/200 计数，超长会被截断）——生成内容时控制长度。
