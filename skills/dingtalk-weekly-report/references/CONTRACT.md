# 输入、缺失处理与输出契约

## 用户必须提供或确认

| 阶段 | 输入 | 未提供或不合法时 |
|---|---|---|
| 首次配置 | 姓名、表单项目完整原文、附件项目 | `config.json` 校验失败并列出字段；不登录、不填表 |
| 目标周 | 可选的周一日期 | 周一默认上一周；其他日期默认本周；执行前仍由用户确认 |
| 工作内容 | 工作日志路径，或逐日口述内容 | 日志留空时生成 TODO 骨架并进入访谈；已配置但文件失效时阻断 |
| 登录 | 有效登录态，或内部二维码 auth 链接/扫码 | 阻断并要求用户重新登录；auth URL 必须来自 `https://*.h3yun.com/entry/auth` |
| 人工审核 | 逐日内容、工时合计、同周旧草稿检查 | 不得使用 `--draft --confirmed`；工具没有提交能力 |

`config.json` 的完整字段模板见 `assets/config.example.json`。`progress_report` 是唯一可选的业务输入；
没有工作日志不等于可以自动编造内容，必须由用户逐日补充。

## 中间产物与最终输出

| 产物 | 路径/位置 | 用途 |
|---|---|---|
| 周报 JSON | `$WORK/weeks/week_report_YYYYMMDD.json` | 人审锚点；已存在时拒绝覆盖 |
| 附件 | `$WORK/output/YYYYMMDD-YYYYMMDD本周工作总结与下周计划.xlsx` | 上传到周报表单 |
| 填表预览 | `$WORK/output/shots/20-filled-review.png` | 暂存前核对字段、行数、工时 |
| 暂存结果 | `$WORK/output/shots/30-saved.png` + 氚云草稿 | 工具可到达的最终状态 |
| 登录态 | `~/.config/dtwr/state.json`，权限 0600 | 凭证，不得入库或转发 |

最终“提交”不属于工具输出：用户必须在钉钉中打开草稿、再次核对并亲手提交。
`--confirmed` 只是操作者已完成人审和旧草稿检查的声明，不提供身份认证、不可抵赖或审计证明。

## 失败原则

- 配置占位值、TODO、空必填项、错误周次、工作日缺失、非法枚举、工时异常、内容超过 200 字：
  在生成附件或填表前阻断，并一次列出全部问题。
- 项目下拉无法精确选中、附件不存在、暂存后未观察到成功提示：
  返回失败并保留 `99-error.png`，不声称成功。
- 同周旧草稿目前无法可靠自动识别：由用户检查，`--confirmed` 是显式确认。
- `$WORK`/登录态属主自动检查仅在 POSIX 系统启用；Windows 依赖用户目录 ACL，验收时需人工确认账户隔离。
